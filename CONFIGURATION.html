üíæ Database Configuration

DATABASE_PATH

DATABASE_PATH=./db/apd_detection.db

Description: Path ke file database SQLite.

Options: - Relative path: ./db/apd_detection.db - Absolute path: /var/data/apd_detection.db - In-memory (testing): :memory:

Recommendations: - Development: ./db/apd_detection.db - Production: /var/lib/apd/apd_detection.db (dengan proper permissions)

Notes: - Directory harus sudah ada atau writable - Database akan dibuat otomatis jika belum ada - Backup database secara berkala untuk production

üéØ Detection Settings

CONFIDENCE_THRESHOLD

CONFIDENCE_THRESHOLD=0.5

Description: Minimum confidence score untuk deteksi valid.

Range: 0.1 - 1.0

Recommendations: - High Precision (sedikit false positive): 0.6 - 0.7 - Balanced: 0.5 (recommended) - High Recall (detect lebih banyak): 0.3 - 0.4

Impact: - Nilai tinggi ‚Üí Lebih sedikit deteksi, lebih akurat - Nilai rendah ‚Üí Lebih banyak deteksi, lebih banyak false positive

FPS

FPS=10

Description: Target frame per second untuk processing.

Range: 1 - 60

Recommendations: - High-end PC/GPU: 30 - Mid-range PC: 15 - 20 - Low-end PC/CPU only: 5 - 10 - Real-time critical: 30 - Review purposes: 10

Impact: - FPS tinggi ‚Üí Lebih smooth tapi lebih berat CPU/GPU - FPS rendah ‚Üí Lebih hemat resource tapi kurang smooth

VIDEO_FRAME_SAMPLE_RATE

VIDEO_FRAME_SAMPLE_RATE=30

Description: Save detection setiap N frames (untuk mengurangi redundansi di database).

Range: 1 - 120

Recommendations: - 30 fps video: 30 (save setiap 1 detik) - 60 fps video: 60 (save setiap 1 detik) - Low fps video: 10 (save lebih sering)

Example: - Video 30 fps, sample rate 30 ‚Üí Save 1 detection per detik - Video 60 fps, sample rate 60 ‚Üí Save 1 detection per detik

VIDEO_SAVE_ONLY_ON_CHANGE

VIDEO_SAVE_ONLY_ON_CHANGE=true

Description: Hanya save ke database jika ada perubahan signifikan.

Values: true / false

Recommendations: - Production: true (menghemat storage) - Development/Debug: false (save semua)

VIDEO_CHANGE_THRESHOLD

VIDEO_CHANGE_THRESHOLD=20.0

Description: Minimum perubahan (%) untuk dianggap ‚Äúsignificant change‚Äù.

Range: 5.0 - 50.0

Recommendations: - Sensitive (save banyak perubahan): 10.0 - Balanced: 20.0 (recommended) - Conservative (hanya perubahan besar): 30.0

Example: - Threshold 20% ‚Üí Save jika deteksi berubah ‚â•20% - Scene dengan 5 orang ‚Üí threshold 1 orang (20% dari 5)

üîî Notification Settings

ENABLE_TELEGRAM

ENABLE_TELEGRAM=false

Description: Enable/disable Telegram notifications.

Values: true / false

Requirements untuk true: - TELEGRAM_BOT_TOKEN harus diisi - TELEGRAM_CHAT_ID harus diisi - Internet connection tersedia

TELEGRAM_BOT_TOKEN

TELEGRAM_BOT_TOKEN="123456789:ABCdefGHI..."

Description: Token dari Telegram BotFather.

How to get: 1. Buka Telegram, cari @BotFather 2. Send /newbot 3. Follow instruksi 4. Copy token yang diberikan

Format: <bot_id>:<token_string>

Example: "1234567890:AABBCCDDEEFFGGHHIIJJKKLLMMNNOOPPqqTT"

TELEGRAM_CHAT_ID

TELEGRAM_CHAT_ID="123456789"

Description: Chat ID tujuan notifikasi.

How to get:

Personal Chat: 1. Cari @userinfobot di Telegram 2. Send any message 3. Copy your ID

Group Chat: 1. Add @getidsbot ke group 2. Copy group ID (starts with -)

Format: - Personal: 123456789 - Group: -123456789

NOTIFICATION_COOLDOWN_SECONDS

NOTIFICATION_COOLDOWN_SECONDS=300

Description: Cooldown period untuk mencegah spam notifikasi (dalam detik).

Range: 60 - 3600 (1 menit - 1 jam)

Recommendations: - High traffic area: 300 (5 menit) - Medium traffic: 180 (3 menit) - Low traffic: 120 (2 menit) - Urgent scenarios: 60 (1 menit)

Behavior: - Person yang sama dengan violation yang sama akan di-cooldown - Setelah cooldown expired, notifikasi akan dikirim lagi

MIN_SEVERITY_FOR_NOTIFICATION

MIN_SEVERITY_FOR_NOTIFICATION=MEDIUM

Description: Minimum severity level untuk trigger notifikasi.

Values: - LOW - Notify semua violations - MEDIUM - Notify MEDIUM, HIGH, CRITICAL - HIGH - Notify HIGH, CRITICAL only - CRITICAL - Notify CRITICAL only

Recommendations: - Production: MEDIUM atau HIGH - Testing: LOW - Critical areas: HIGH

üö® Violation Settings

VIOLATION_COOLDOWN_SECONDS

VIOLATION_COOLDOWN_SECONDS=3

Description: Cooldown untuk violation detection (mencegah duplicate violations dari frame yang berurutan).

Range: 1 - 10 detik

Recommendations: - Fast-moving workers: 2 - 3 - Slow-moving workers: 5 - Static monitoring: 10

Impact: - Cooldown rendah ‚Üí Lebih banyak violation records - Cooldown tinggi ‚Üí Lebih sedikit duplicate violations

VIOLATION_TRACK_PER_PERSON

VIOLATION_TRACK_PER_PERSON=true

Description: Track violation per individual person (menggunakan ByteTrack ID).

Values: true / false

Benefits if true: - Cooldown per-person (lebih akurat) - Bisa track violation history per individu - Better notification management

Drawbacks if false: - Global cooldown untuk semua orang - Tidak bisa distinguish individual violations

Recommendations: Always true untuk production

üåç Location & Camera Settings

LOCATION

LOCATION=Terminal Pelabuhan A

Description: Nama lokasi untuk record keeping.

Format: Any string (max 255 characters)

Examples: - Terminal Pelabuhan A - Workshop Area 2 - Main Gate - North Entrance - Construction Site - Building 3

CAMERA_ID

CAMERA_ID=CAM-001

Description: Identifier kamera untuk multi-camera setup.

Format: Any string (recommended: alphanumeric with dash)

Examples: - CAM-001, CAM-002, ‚Ä¶ - GATE-NORTH-01 - WORKSHOP-A-MAIN

TIMEZONE

TIMEZONE=Asia/Surabaya

Description: Timezone untuk timestamp.

Format: IANA timezone database format

Common Values: - Asia/Jakarta (WIB - GMT+7) - Asia/Makassar (WITA - GMT+8) - Asia/Jayapura (WIT - GMT+9) - UTC (Universal Time)

Find your timezone: https://en.wikipedia.org/wiki/List_of_tz_database_time_zones

üîê Security Best Practices

1. Environment File Protection

# Set proper permissions
chmod 600 python-script/.env

# Never commit .env to git
echo ".env" >> .gitignore

2. Telegram Token Security

  • ‚ùå Never commit token ke Git
  • ‚ùå Never share token publicly
  • ‚úÖ Use environment variables
  • ‚úÖ Rotate token periodically

3. Database Security

# Set proper permissions
chmod 644 db/apd_detection.db
chmod 755 db/

# Regular backups
cp db/apd_detection.db db/backups/apd_detection_$(date +%Y%m%d).db
